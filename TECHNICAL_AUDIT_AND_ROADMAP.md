## 5. Deep Audit Findings (March 19, 2026)

### 5.1 Architecture & Core Systems
- **Routing:** Overlapping routes in `App.tsx` (`/teacher/*` vs `/teacher/studio`). Needs refactoring to use nested React Router v6 layouts properly.
- **State Management:** `SessionContext.tsx` handles too much monolithic logic. Needs to be split into domain-specific stores (e.g., using Zustand as originally planned).

### 5.2 Teacher & Student Flows
- **Drawing Synchronization:** The `DrawingLayer` in teacher remote updates local state in `SessionContext.tsx` but `startDrawing`, `addDrawingPoint`, etc., do not emit socket events. (High Priority)
- **Spaced Repetition:** The SRS UI is implemented, but relies entirely on `MockEngine`. Requires a real Leitner system or SuperMemo-2 algorithm in the backend.

### 5.3 UX & Polish
- **Board Transitions:** `ClassroomBoard.tsx` replaces templates instantly via a React `key` change and CSS fade. Needs `framer-motion` `AnimatePresence` for smooth crossfades and sliding animations.
- **Asset Generation:** The AI Asset Enrichment in Lesson Studio relies on `picsum.photos` or mock image URLs. Needs true integration with Google Imagen or similar APIs.

## 6. Pending Ecosystem Refinements (Immediate Focus)

## 7. Backend & Database Integration Plan (Phase 3)

*To be executed ONLY after Phase 1 and 2 are 100% complete.*
Once the ecosystem refinements are complete, the product will transition from the `MockEngine` to a production-ready Supabase backend.

### Step 1: Database Setup
- Initialize Supabase project and execute `supabase_schema.sql` to create `organizations`, `users`, `classes`, `units`, `lesson_blocks`, and `game_assets` tables.
- Configure Row Level Security (RLS) policies for multi-tenant isolation.

### Step 2: Storage & Realtime
- Implement Supabase Storage buckets for uploading textbook PDFs and saving AI-generated audio/image assets.
- Replace Express `socket.io` server with Supabase Realtime channels for classroom state syncing (`classroom_action` payloads).

### Step 3: API & Auth Refactoring
- Replace all `MockEngine` calls in `SessionContext.tsx` and teacher dashboard components with actual Supabase client queries.
- Implement serverless edge functions for secure Gemini API calls (to avoid exposing the API key on the client).
- Implement robust Authentication flows for Teacher, Student, and Parent roles.
